# 野人先生 AI 设备维修协同中枢 — 实施计划

> 交付类型：本地可运行中文 Web Demo V1  
> 数据边界：全部门店、设备、责任方、SLA 与案例均为 Demo 模拟数据。

## 阶段与验收

| 阶段 | 交付内容 | 完成标准 |
|---|---|---|
| 1. 骨架与数据 | Next.js、TypeScript、Prisma、SQLite、Schema、种子与重置脚本 | 一键初始化并获得 3 店、9 设备、6 责任方、10 规则、20 事件 |
| 2. 角色与页面 | 四角色切换、公共详情、门店/管理/供应商/运营路由 | 需求列出的路由均可访问，中文企业级界面 |
| 3. AI 与路由 | 确定性 AIProvider、Zod Schema、完整性检查、可解释规则路由 | 正常、缺信息、低置信度、高风险稳定分流 |
| 4. 工单与 SLA | 状态机、权限校验、接拒单、处理、超时与通知 | 每次合法状态变化写 StateEvent，非法操作被阻止 |
| 5. 验收与履历 | 维修结果、验收、返修、关闭、设备历史 | 未验收禁止关闭，关闭后设备页可追溯 |
| 6. 测试与迁移 | 单元/集成测试、飞书 CSV、映射、Demo 脚本、README | lint、typecheck、test、build 全部通过 |

## 技术方案

- Next.js App Router + TypeScript；
- Prisma + 本地 SQLite；
- Zod 约束输入与 AI 输出；
- 确定性的 `AIProvider` 本地实现，不依赖外部模型；
- Recharts 轻量图表；
- Vitest 单元与集成流程测试；
- API Route 统一执行权限、状态、日志和数据写入。

## 核心演示顺序

正常自动派单 → 信息缺失补充 → 低置信度人工复核 → P1 超时升级 → 验收不通过返修 → 验收通过关闭并查看设备履历。
