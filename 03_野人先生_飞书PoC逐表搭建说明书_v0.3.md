# 1. PoC搭建目标

# 本次飞书PoC不是简单做一张工单表，而是需要真实跑通以下闭环：

```text
门店提交报修
→ AI分析原始描述
→ 判断信息是否完整
→ 信息不足则要求补充
→ 信息完整后执行责任路由
→ 低置信度或规则冲突进入人工复核
→ 创建并派发统一工单
→ 供应商或维修人员接单
→ 远程处理或预约上门
→ 提交维修结果
→ 门店验收
→ 返修或关闭
→ 形成设备维修履历
```

# PoC完成后，至少应当能够真实演示：

1. 正常自动派单；
2. 报修信息缺失并自动补充；
3. AI低置信度转人工；
4. 供应商超时未接单并升级；
5. 门店验收不通过并返修；
6. 工单关闭后形成设备履历。

# 2. 多维表格总体结构

# 建议在一个飞书多维表格应用中建立以下11张核心数据表：

| **编号** | **数据表** | **用途**               |
| ------ | ------- | -------------------- |
| T01    | 门店表     | 存储门店、区域、联系人及门店类型     |
| T02    | 设备表     | 存储设备身份、所属门店、供应商和保修信息 |
| T03    | 供应商表    | 存储供应商服务范围、区域和接单负责人   |
| T04    | 故障事件表   | 保留门店最初提交的原始报修信息      |
| T05    | AI分析表   | 保存每一次AI分析结果和版本       |
| T06    | 路由规则表   | 配置责任方、紧急度和SLA规则      |
| T07    | 维修工单表   | 作为三方协同的唯一主记录         |
| T08    | 状态事件表   | 记录每次状态和责任方变化         |
| T09    | 预约记录表   | 记录远程处理或上门预约          |
| T10    | 维修记录表   | 记录实际故障原因和维修结果        |
| T11    | 验收记录表   | 记录门店验收、返修和关闭结论       |

# 建议另外建立两张辅助表：

| **编号** | **数据表** | **用途**           |
| ------ | ------- | ---------------- |
| A01    | 字典配置表   | 管理设备类别、故障分类和风险标签 |
| A02    | 消息记录表   | 记录系统发送过的提醒和升级消息  |

# 首版时间有限时，A01和A02可以先不做。

# 3. 数据表之间的关联关系

```text
门店表
  └── 一对多 ── 设备表
                  ├── 多对一 ── 供应商表
                  └── 一对多 ── 故障事件表
                                    ├── 一对多 ── AI分析表
                                    └── 一对一/多 ── 维修工单表
                                                      ├── 一对多 ── 状态事件表
                                                      ├── 一对多 ── 预约记录表
                                                      ├── 一对多 ── 维修记录表
                                                      └── 一对多 ── 验收记录表

路由规则表
  └── 根据设备、故障、风险、保修、区域
      为维修工单推荐责任方、紧急度和SLA
```

# 核心原则：

> 故障事件保留门店最初提交的事实，AI分析保留模型处理结果，维修工单保留最终业务决策。三类数据不能混在同一个字段中相互覆盖。

* 填写接单时间；

```text
4. 维修动作和维修结果；

你输出的是信息提取和分类建议，不是专业维修诊断。

当输入中出现以下风险时：

- 冒烟；
- 漏电；
- 严重焦味；
- 人员受伤；
- 严重温控失控；
- 可能影响食品安全；

必须：

1. 将requires_human_review设置为true；
2. 标记对应risk_tags；
3. 不提供拆机、接电、复位或制冷剂操作；
4. 只提示停止自行处理并等待专业人员；
5. 优先询问人员是否安全、设备是否停机和现场是否隔离。

输出必须是合法JSON，不添加JSON之外的文字。
```
